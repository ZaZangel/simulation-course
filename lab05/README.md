### Моделирование случайных событий (GUI)

**Часть 1:**  
Приложение «Скажи “да” или “нет”».

**Часть 2:**  
Приложение «Шар предсказаний» (Magic 8-Ball).



## 1. Теоретическая часть

В основе логики приложения лежат два метода генерации случайных событий, основанные на теории вероятностей:

### 1.1. Равномерная выборка (Uniform Distribution)
Используется для случаев, когда все исходы равновероятны.

*   **Формула:** $P(A_i) = \frac{1}{n}$, где $n$ — количество возможных исходов.
*   **Пример:** Для генератора «Да / Нет» ($n=2$):
    $$P(\text{ДА}) = 0.5, \quad P(\text{НЕТ}) = 0.5$$
*   **Реализация:** Обычно реализуется через стандартные функции выбора, но в данном проекте реализована как частный случай взвешенного алгоритма с равными весами.

### 1.2. Взвешенный случайный выбор (Алгоритм Моисеева)
Используется для случаев, когда исходы имеют разную вероятность появления.

*   **Входные данные:** Набор событий $A_k$ с вероятностями $P_k$, где $\sum P_k = 1$.
*   **Генератор:** $\xi \sim U[0, 1)$ — равномерно распределённая случайная величина.
*   **Логика алгоритма:** Последовательная проверка накопленной вероятности. В классической формулировке происходит вычитание вероятностей до тех пор, пока остаток не станет меньше или равен сгенерированному случайному числу.
    
    *Примечание:* В коде реализована эквивалентная вариация этого алгоритма через **накопительную сумму** (Inverse Transform Sampling), где проверяется условие попадания случайного числа в интервал накопленных вероятностей.

## 2. Анализ реализации

### 2.1. Единая функция выбора
Вся логика вероятностей инкапсулирована в одной функции, что соответствует теоретическому описанию взвешенного выбора.

**Компоненты функции:**
*   **Генератор случайного числа:** Используется стандартный генератор, обеспечивающий равномерное распределение $\xi$ на интервале $[0, 1)$.
*   **Накопление вероятностей:** Последовательно суммируются вероятности каждого события.
*   **Проверка условия:** Определяется момент, когда накопленная сумма становится больше или равна случайному числу, что указывает на выбранное событие.

### 2.2. Вкладка 1: Котик (Равномерная выборка)
Здесь теория равномерной выборки реализована через взвешенную функцию с равными коэффициентами.

**Набор ответов:**
*   ДА — вероятность 0.5
*   НЕТ — вероятность 0.5

**Соответствие теории:** Так как вероятности равны, это частный случай равномерного распределения, где $P(A_i) = 1/n$. Интервалы для «ДА» и «НЕТ» составляют $[0, 0.5]$ и $(0.5, 1.0]$ соответственно.

### 2.3. Вкладка 2: Magic Ball (Взвешенная выборка)
Здесь реализован полный алгоритм Моисеева (в вариации накопительной суммы) с разными вероятностями.

**Набор ответов и их вероятности:**
| Ответ | Вероятность |
|:------|:-----------:|
| Да | 0.15 |
| Нет | 0.15 |
| Скорее всего | 0.10 |
| Сомнительно | 0.10 |
| Без сомнений | 0.10 |
| Спроси позже | 0.15 |
| Определенно да | 0.10 |
| Маловероятно | 0.15 |

**Соответствие теории:**
*   Сумма вероятностей равна 1.0, что обеспечивает корректную работу алгоритма.
*   События имеют разные веса (0.1 против 0.15), что создаёт неравномерное распределение исходов.
*   При генерации случайного числа $\xi$ шанс выпадения ответа с вероятностью 0.15 выше, чем шанс выпадения ответа с вероятностью 0.1.

### 2.4. Визуализация теории

**Анимация процесса:**
Пока происходит вычисление (выбор события), пользователь видит анимацию — моргание котика в первой вкладке или вращение шара во второй. Это скрывает дискретность мгновенного вычисления случайного числа и создаёт иллюзию «процесса гадания», что повышает вовлечённость пользователя.

**Графические эффекты:**
Вторая вкладка использует систему частиц, где начальные координаты, скорости и направления движения также генерируются случайным образом с использованием равномерного распределения. Это расширяет применение теории вероятностей на визуальную составляющую приложения.

## 3. Вывод

Приложение корректно реализует заявленные теоретические методы генерации случайных событий.

1.  **Универсальность подхода:**
    Вместо использования разных функций для равномерного и взвешенного выбора, приложение использует единый алгоритм взвешенного выбора, который при равных весах вырождается в равномерное распределение. Это демонстрирует математическую корректность реализации.

2.  **Качество генерации:**
    Используется стандартный генератор псевдослучайных чисел, обеспечивающий величину $\xi \sim U[0,1)$. Данный генератор обладает достаточными статистическими свойствами для целей приложения.

3.  **Гибкость и масштабируемость:**
    Вероятности вынесены в отдельные структуры данных, что позволяет легко изменять теоретическое распределение вероятностей без необходимости модификации основной логики алгоритма. Это обеспечивает простоту настройки и расширения функциональности.

4.  **Пользовательский опыт:**
    Анимационные эффекты и визуализация процесса выбора скрывают математическую природу алгоритма, создавая у пользователя ощущение интерактивного гадания, что соответствует цели приложения.
